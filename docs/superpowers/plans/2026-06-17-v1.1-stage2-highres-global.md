# V1.1 Stage2 全局高分重建训练 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (inline execution with checkpoints) or superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 V1.1 AEF 蒸馏模型权重基础上，加入高分光学/SAR 重建目标并在 harbin+haidian 全量 744 patch 上继续训练，得到 Stage2 checkpoint；重新提取 202512 embedding 后跑 4-task 5-fold 下游 benchmark，生成并发布 V1.1-Stage2 vs AEF 2025 对比报告。

**Architecture:** 复用已有的 `MonthlyEmbeddingDataset` / `AEFModel` / `TotalLoss` 流水线，仅通过新增 `highres_*_recon` target_heads 启用高分重建监督；使用 `--init-from` 从 V1.1 `best.pt` 加载模型权重并随机初始化新增的高分 decoder；训练配置降低学习率与蒸馏权重，优先让高分重建头收敛。全程复用已有的预处理缓存，避免重新生成大体积缓存。

**Tech Stack:** PyTorch / torch_npu, YAML 配置, `scripts/train/train.py`, `scripts/train/launch_6card.sh`, `downstreams/scripts/precompute_embeddings.py`, `downstreams/scripts/train_task.py`, 现有报告生成脚本。

### Task 5: 4-task下游 benchmark

**Files:**
- Create: `scripts/eval/run_v1.1_stage2_benchmark_6card.sh`
- Output: `/data/xuannv_embedding/experiments/v1.1_long/v1.1_distill_long_stage2_benchmark/`

- [ ] **Step 1: 创建 benchmark 脚本**

```bash
#!/bin/bash
set -e

# 6× NPU 并行版 4-task × 5-fold benchmark for V1.1 Stage2.
# Usage: scripts/eval/run_v1.1_stage2_benchmark_6card.sh <checkpoint_path> <output_root> [embedding_root]

CHECKPOINT="${1:?请提供 checkpoint 路径}"
OUTPUT_ROOT="${2:?请提供输出根目录}"
EMB_ROOT="${3:-}"

WORKTREE="/root/workspace/xuannv/.worktrees/feat-multitask-downstream"
cd "$WORKTREE"

mkdir -p "$OUTPUT_ROOT"

if [[ -z "$EMB_ROOT" ]]; then
  echo "==> Extracting v1.1 Stage2 embeddings for all labeled patches ..."
  python downstreams/scripts/precompute_embeddings.py \
    --config configs/v1.1_extract_labeled_all_stage2.yaml \
    --checkpoint "$CHECKPOINT" \
    --regions harbin haidian \
    --output-root /data/xuannv_embedding/embeddings/v1.1_labeled
  EMB_ROOT=$(ls -td /data/xuannv_embedding/embeddings/v1.1_labeled/*/ | head -1)
  EMB_ROOT="${EMB_ROOT%/}"
fi

echo "==> Embeddings: $EMB_ROOT"

echo "==> Running 4 harbin tasks in parallel on NPU 0-3 ..."
pids=()
tasks=(construction building_change farm_change rubbish)
for i in "${!tasks[@]}"; do
  task="${tasks[$i]}"
  mkdir -p "$OUTPUT_ROOT/$task"
  ASCEND_RT_VISIBLE_DEVICES="$i" python downstreams/scripts/train_task.py \
    --config "downstreams/configs/aef_benchmark/${task}.yaml" \
    --task "$task" \
    --embedding-root "$EMB_ROOT" \
    --label-root /data/xuannv_embedding/processed/harbin/labels \
    --output-root "$OUTPUT_ROOT/$task" > "$OUTPUT_ROOT/${task}.log" 2>&1 &
  pids+=("$!")
  echo "  $task -> NPU $i, PID ${pids[-1]}"
done

fail=0
for i in "${!pids[@]}"; do
  if ! wait "${pids[$i]}"; then
    echo "任务 ${tasks[$i]} (PID ${pids[$i]}) 失败"
    fail=1
  fi
done
if [[ "$fail" -ne 0 ]]; then
  echo "有下游任务失败，退出"
  exit 1
fi

echo "==> Generating V1.1-Stage2 vs AEF 2025 comparison report/figure ..."
python scripts/eval/generate_v1.1_vs_aef_report.py \
  --v11-root "$OUTPUT_ROOT" \
  --output-dir /root/workspace/report/v1.1_stage2_benchmark \
  --tasks construction,building_change,farm_change,rubbish

echo "==> Done. Reports:"
echo "  /root/workspace/report/v1.1_stage2_benchmark/V1.1_vs_AEF_REPORT.md"
```
