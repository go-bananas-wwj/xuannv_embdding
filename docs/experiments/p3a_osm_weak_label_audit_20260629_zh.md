# P3A.0 OSM 弱监督标签审计

- 生成时间：2026-06-29
- 数据输出：`/data/xuannv_embedding/experiments/p3a_osm_audit_20260629`
- 审计对象：海淀、哈尔滨现有 `building_osm` / `road_osm` 的 OSM GeoPackage 缓存和 128x128 栅格 mask。
- 使用原则：OSM 只能作为弱监督语义先验，不能把 OSM 空白区域当强背景负样本。

## 核心结论

当前 OSM 缓存不能直接支持“月度变化真值”训练。

原因很明确：

1. 只有海淀建筑缓存里有少量时间字段，覆盖率极低。
2. 海淀道路、哈尔滨建筑、哈尔滨道路基本没有可用时间字段。
3. OSM 的编辑时间或少量 `check_date` 只能说明地图何时被编辑/核查，不能代表真实世界这个月发生建设或拆除。
4. 当前缓存更适合作为建筑、道路、城市结构的弱语义先验，用来训练 density / presence / edge 这类软目标。

## 时间字段细节

海淀建筑：

- `start_date`：45 条，占 0.10%。里面有 `1958-04-22`、`1771`、`2023-09-28`、`2003竣工` 这类值，格式不统一。
- `opening_date`：1 条，占 0.00%，值为 `2025-10-18`。
- `construction_date`：11 条，占 0.02%，多数是历史建筑年份。
- `check_date`：12 条，占 0.03%，可以作为近期核查置信度，但不能当建设时间。
- `disused:amenity`：2 条，占 0.00%。

其他三个缓存没有可用时间字段。

## 对训练的含义

可以做：

1. 用 OSM 建筑生成建筑密度图，而不是硬二值真值。
2. 用 OSM 道路生成道路密度图、中心线或边缘辅助图。
3. 对 OSM 标签设置 confidence mask，降低缺失和错位影响。
4. 对建筑、道路这种相对稳定目标做弱语义约束。

暂时不要做：

1. 不要用当前 OSM 缓存监督 2025-12 到 2026-05 的月度变化。
2. 不要用 OSM 空白作为强负样本。
3. 不要把少量 `start_date` / `check_date` 当作真实变化月份。
4. 不要把下游任务标注混进主 embedding 训练。

## P3A 下一步建议

我建议 P3A.1 先做“月度状态重建增强”，不等待 OSM 历史数据：

1. 新增 NDVI / NDWI / NDBI / BSI 当月重建目标。
2. 新增跨月 `ΔNDVI / ΔNDWI / ΔNDBI / ΔBSI` 变化重建目标。
3. 保持单一月度 embedding，不做双通道。
4. OSM 暂时只作为弱语义 density 辅助目标，权重小一些。
5. 后台并行准备 OSM full-history `.osh.pbf` 或月度历史快照下载与解析，再决定是否加入真正的 OSM 历史监督。

## 结论

当前数据支持我们进入 P3A.1，但 OSM 的角色必须收敛：

**OSM 当前缓存 = 弱语义先验。**

**OSM 历史变化 = 需要重新下载 full-history 或月度快照后再审计。**

所以下一轮主线应该先做月度状态重建和跨月指数变化重建，把 embedding 训练成“每个月真实地表状态”的表达；OSM 只在旁边轻轻拉一下建筑和道路语义，不让它主导模型。
