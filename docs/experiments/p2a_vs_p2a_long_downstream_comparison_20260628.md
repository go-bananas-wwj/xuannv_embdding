# P2A vs P2A-Long 下游任务指标对比（2026-06-28）

本文档对比 P2A 原版与 P2A-Long 在变化检测和 OSM 下游任务上的 quick fold-0 结果。P2A-Long 使用 P2A best checkpoint 继续训练 100 epoch，学习率 5e-5。

## 结论

- P2A-Long 的训练验证损失更低，但下游任务没有整体变好。
- 变化检测平均 F1 从 0.3394 降到 0.3049，下降 0.0345。
- OSM Linear 平均 F1 基本持平：0.2463 到 0.2467。
- OSM MLP 平均 F1 从 0.3403 降到 0.3100，下降 0.0303。
- OSM U-Net 平均 F1 从 0.5077 降到 0.4646，下降 0.0431。
- 因此当前不建议用 P2A-Long 替换 P2A，P2A 仍然是更合适的升级底座。

## 任务级差异解读

- Linear probe 基本没有变差，海淀建筑和海淀道路略升，但哈尔滨建筑下降，说明继续训练没有稳定提升线性可分性。
- MLP probe 除海淀道路略升外，其余任务下降，尤其哈尔滨建筑从 0.4044 降到 0.2977。
- U-Net probe 四个 OSM 任务全部下降，说明更强下游头也没有从 P2A-Long embedding 中获得更多有效空间语义。
- 这类现象通常意味着继续优化当前自监督/辅助训练目标，会让模型更贴合训练目标本身，但不一定更适合建筑、道路、农田变化等真实下游任务。

## 后续建议

1. 保留 P2A 原版作为当前 baseline。
2. 不继续单纯加训练轮次。
3. 下一轮优先改训练目标：引入更强的语义区域约束、变化区域对比、边界/细目标友好的道路建筑监督信号。
4. 每轮训练后继续固定执行：导出 embedding、变化检测 quick eval、OSM Linear/MLP/U-Net eval、红白预测可视化、与 P2A 对比。
